Perguntas sobre o Git

Para que serve o commit e qual a melhor forma de utilizá-lo?

De forma mais ampla, em ciência da computação, um commit é uma confirmação de alterações feitas, marcando o fim de uma transação.

Em um sistema de versionamento de código, como nosso exemplo o Git, o commit confirma e faz uma marcação para todas as alterações adicionadas à ele, gerando um histórico de commits e sendo possível fazer um checkout em cada um.

Não existe somente uma melhor forma de utilizar o commit, mas existem boas práticas, as quais são muito comuns no mundo do desenvolvimento de software. De tão importante que são as boas práticas que foi criado o Conventional Commits , uma iniciativa que tenta oficializar a forma correta de se fazer commits.

Como boa prática, o commit deve ser feito sempre que um trecho de código for feito e estiver funcionando. Ao fazer esse commit ele deve conter uma mensagem auto explicativa do que foi feito naquela versão. Seguindo o Conventional Commits, essas mensagens devem seguir de um prefixo, que identifique se é uma correção (fix:), nova funcionalidade (feat:) e até correção em produção (hotfix:).
