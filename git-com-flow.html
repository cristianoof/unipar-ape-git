Contextualizando um pouco o Git Flow

Git Flow é uma estratégia ou metodologia de trabalho, ela foi criada para facilitar o trabalho em equipe com o Git.

Com o Git flow, trabalhamos com 5 branches: Master / Main, Develop, Feature, Release e Hotfix. Essa divisão em branches é o que traz toda a organização para o projeto, onde a Master está o código em produção, a Develop a próxima versão candidata a produção, Feature onde estão sendo desenvolvidas as novas funcionalidades, Release que faz a ponte entre a Develop e Master e Hotfix para correções.

Temos duas maneiras de trabalhar com o Git Flow: uma com os comandos tradicionais do Git, porém seguindo metodologia do Git Flow e outra é instalando a CLI do Git Flow e usando os comandos específicos da CLI.

Abaixo vamos abordar os principais comandos utilizando a CLI do Git Flow

Comando Descrição
$ git flow init O comando cria a branch Develop com base na Master.
$ git flow feature start [nome-feature] O comando cria a branch Feature
$ git flow feature finish [nome-feature] Ao finalizar uma funcionalidade desenvolvida na Feature, esse comando faz a finalização
$ git flow hotfix start [nome-hotfix] O comando cria a branch Hotfix
$ git flow hotfix finish [nome-hotfix] Ao finalizar uma correção desenvolvida em Hotfix, esse comando faz a finalização
$ git flow release start [1.0.0] O comando cria a branch Release, nesse caso incluindo a versão
$ git flow release finish [1.0.0] Ao finalizar o desenvolvimento na Release, esse comando faz a finalização
$ git flow feature pull [nome-feature] Puxa os arquivos atualizados do repositório removo para o repositório local, nesse exemplo usamos a branch Feature, mas o comando é o mesmo para as outras branches
